# 固定测试结果发布规范

## 目标

StarTest 只记录 Star 系列软件可从外部观察到的功能和性能。案例目录、
案例 ID、真值模型与验收单位保持稳定；算法版本变化通过 `runs/` 下的新
运行目录比较，不再按日期创建另一套测试树。

## 允许发布

- 信号类型、PRN、采样率、时长和随机种子；
- C/N0 时间表、载波多普勒真值、TCXO模型和捕获移交误差；
- StarTrack Git SHA、公开档案版本、运行时刻和配置哈希；
- 估计 C/N0、跟踪多普勒、动态码相位、状态和异常事件；
- 预期结果、实际指标、通过或失败结论；
- 从上述黑盒数据绘制的图表。

## 不发布

- 环路带宽、滤波器内部状态和过程噪声；
- 鉴频、鉴相、鉴码公式及其中间变量；
- 原子相关值、观察窗内容和相关器抽头；
- 私有配置、源码和20 ms调试日志；
- 能够直接反推出算法实现的内部诊断量。

## 固定案例

每个信号必须包含以下八类固定案例：

1. 牵引灵敏度。
2. 无电文辅助持续跟踪。
3. 已知电文辅助持续跟踪。
4. 多普勒动态能力。
5. 强弱阶跃切换。
6. 缓慢遮挡与恢复。
7. 状态门限徘徊。
8. 一小时耐久测试。

案例 `README.md` 必须包含现实场景、输入、真值、预期结果、实际结果和
结论。`scenario.json` 保存稳定案例 ID；日期只允许出现在运行目录内的
`summary.json` 元数据中。

## 运行目录

运行目录命名为：

```text
startrack-<git-sha>_<profile-version>
```

每个运行必须提供：

- `summary.json`：Git SHA、档案版本、运行时刻、案例 ID 和种子；
- `metrics.csv`：逐种子的状态、频差和码相位指标，字段名携带单位；
- `seed-NNN/observations.csv`：降采样到1 Hz的公开观测；
- `seed-NNN/state_events.csv`：状态及异常事件；
- `figures/*.png`：真值、误差、状态和种子分布图。

20 ms 原始日志仅保存在本地 Development 工作区。

## 统一验收

- 不重捕，不丢同步；
- 载波有效观测率不低于95%；
- 多普勒 RMS 不超过5 Hz，P95不超过10 Hz；
- 动态真实码相位 P95不超过0.20 chip；
- 状态转换延迟从真值越过门限算至硬件命令确认。

灵敏度定义为名义 `GOOD_TCXO_V1` 条件下5个固定种子全部通过的最低
C/N0；下一低1 dB档至少补测一次。功率按有效噪声密度
`-170 dBm/Hz` 换算。
