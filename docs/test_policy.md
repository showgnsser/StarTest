# 测试结果发布规范

## 允许发布

- 信号类型、PRN、采样率、时长、随机种子；
- C/N0 时间表、载波多普勒模型和捕获移交误差；
- 运行耗时、最终状态和状态事件；
- 估计 C/N0、频率误差、码相位误差和锁定标志；
- 预期结果、实际结果、通过或失败结论；
- 从上述黑盒数据绘制的图片。

## 禁止发布

- 软件内部环路带宽、权重和滤波器状态；
- 鉴频、鉴相、鉴码公式及其中间变量；
- 原子相关值、内部观察窗内容和相关器抽头；
- 软件源码、私有配置档案和调试日志；
- 可用于反推出算法实现的内部元数据。

## 结果分层

每个测试集按软件、信号和场景类别归档。每个场景目录包含：

- `scenario.json`：黑盒输入和验收条件；
- `observations.csv`：1 Hz 脱敏外部观测；
- `state_events.csv`：状态和异常事件；
- `result.md`：预期、实际和结论；
- `tracking.png`：C/N0、频率、码相位和状态图。

原始 `.meta.txt`、`.loop.csv`、`.corr.csv` 和软件 stderr 只保存在本地测试工作区。
